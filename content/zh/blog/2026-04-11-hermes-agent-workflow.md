我这次把 Hermes Agent 和博客工作流重新配了一遍。做法不复杂，主要是四步：先记住长期规则，再固定执行时间，然后处理素材不足的情况，最后把发布流程锁定下来。

![Hermes Agent 和博客工作流的概念插图](https://img.freepik.com/premium-vector/ai-coding-robot-assistant-programming-code-debugging-vector-illustration-concept_694862-52.jpg?w=1800)

## 先把长期规则写进去

我先让代理记住几条长期有效的偏好：

- 回答要快，内容要短
- 写作时偏实用，不要空讲概念
- 博客要输出中、英、日三语
- 文章结构不要写死，要按主题调整
- 发布时要走固定的 Git 流程

这一步很重要。只靠一次对话，代理很容易忘掉；写进记忆后，后面的执行才会稳定。

## 再把执行时间固定下来

接下来我给它加了定时任务。

现在的节奏是每周二、四、六自动跑一次。每次执行时，它会先看最近的对话、工作内容和兴趣材料；如果素材够，就直接进入写作；如果素材不够，就先给出几个最近科技和极客方向的选题。

我不希望它在素材不够的时候替我硬写，而是先用选题把我的思考拉出来。这样写出来的内容，才是真正从我自己的判断里长出来的。

## 素材不够时，先做选题，而不是凑稿

这是我这次最在意的地方。

如果当前积累的素材不够，我会让它先从这些方向里找话题：

- AI
- GitHub
- Hugo
- 自动化
- 极客工具链
- 最近的科技热点

然后它不是直接生成成品，而是先给我几个可写的方向，帮助我判断：

- 这个主题有没有继续想下去的价值
- 我自己对这个话题到底在意什么
- 值不值得写成一篇博客

对我来说，这才是写博客真正有意义的地方。

## 发布时走固定的 Git 流程

文章写好以后，我会按固定流程发布：

1. 先同步 main
2. 创建新分支
3. 在分支上写博文
4. push 分支
5. 创建 PR

我希望写作和发布保持清楚边界。草稿就是草稿，正式发布就走正常的协作流程，不把两件事混在一起。

## 我最后确认了什么

这次配置完以后，我确认了两件事：

- Hermes Agent 可以按固定节奏工作
- 素材不足时，它会先帮我找方向，而不是替我硬写

这样一来，博客工作流就不是一次性的设置，而是能持续跑下去的节奏。

最后我想把它总结成一句话：灵活应用 AI 代理的提案能力，让人的输出更有针对性和主动性，最大限度组合 AI 的主动性优势和人的思维创造性。
